## [Unreleased]

- 2026-04-17 (claude) — **Unify status/remarks + enforce NOT NULL** — Dropped duplicate `remarks` column from `copilot_users`; all code now uses `status` exclusively. Normalized `Active` → `access granted` (27 rows). Added `NOT NULL` constraint on `status` with default `'pending'`. Updated `db.js`, `index.html`, `migrate.html`, and `001_schema.sql` to remove all `remarks` references. (refactor/db+code)

- 2026-04-16 (claude) — **Fix inaccurate licensed user and task counts** — Dashboard "Total Licensed Users" now counts only `access granted` users (169) instead of all copilot_users (180). Fixed stale `has_logged_task` boolean for 29 users whose tasks existed but weren't reflected. Updated `practice_summary` view, `get_licensed_tool_adoption` and `get_executive_summary` SQL functions to filter by `status = 'access granted'`. Added default `'pending'` on `copilot_users.status` column. Normalized casing (`Pending` → `pending`). (fix/db+dashboard)

- 2026-04-16 (claude) — **Featured Spotlight Banner + Global Likes System** — New auto-rotating carousel banner at the top of the dashboard showcasing top-performing tasks, accomplishments, prompts, and use cases. Hybrid selection: admin pins first, then user likes, then metric-based fallback. Global permanent like system with heart buttons on Tasks table, Accomplishment cards, Use Case cards, and Prompt Library (reuses existing `prompt_votes`). Banner auto-rotates every 5s with pause-on-hover, manual arrows, dot indicators, and full keyboard/ARIA support. Admin "Banner Settings" page for slot allocation per content type and pin/unpin management. Schema: `likes`, `featured_banner_config`, `featured_banner_pins` tables + `v_banner_candidates` view + `toggle_like` RPC. Client-side daily cache with calendar-day reset. `prefers-reduced-motion` respected. Migration: `sql/022_featured_banner_and_likes.sql`. (feat/ui+db+admin)

- 2026-04-16 (copilot) — **Auto-sync practice_spoc on role change** — When admin assigns a user as SPOC (via modal or inline dropdown), the user is automatically added to `practice_spoc` for their practice. When role changes away from SPOC, their `practice_spoc` entry is deactivated (`is_active = false`). New `syncPracticeSpoc()` in `db.js`. No manual INSERT needed. (feat/approval+db)

- 2026-04-16 (copilot) — **Multi-SPOC Approval per Practice** — Any SPOC in a practice can now approve any task for that practice (first approval wins). Multiple SPOCs can be assigned to the same practice. The "Pending With" column on the employee task status grid, main dashboard approvals page, and admin approvals page now shows actual SPOC usernames instead of a generic "SPOC" label. Schema: removed `UNIQUE(practice)` from `practice_spoc`, added `UNIQUE(practice, spoc_id)`. Updated views `employee_task_approvals`, `pending_approvals`, `spoc_approval_workload` with `pending_spoc_names` column. `fetchPendingApprovals` now matches by practice (not individual `spoc_id`). New `getSpocsForPractice()` returns all active SPOCs. Migration: `sql/021_multi_spoc_approval.sql`. (feat/approval+db+ui)

- 2026-04-16 (copilot) — **Admin Override: Approve Any Task** — Admin can now approve tasks at any approval stage (including `spoc_review`) directly, bypassing the normal SPOC→Admin flow. The state machine in `approveSubmission()` checks `userRole === 'admin'` first and sets `nextStatus = 'approved'` immediately. Dashboard `index.html` now shows Approve/Reject buttons inline for admin on all pending items, and the detail modal `canAct` check allows admin to act on any pending status. (feat/approval)

- 2026-04-16 (copilot) — **Team Lead Role** — New `team_lead` role allowing SPOCs to delegate scoped SPOC-like capabilities to practice contributors. Team leads see only their assigned members' tasks, approvals, and data. Includes: SQL migration `020_team_lead_role.sql` (table `team_lead_assignments`, RLS policies, helper functions), `auth.js` (`isTeamLead()`), `db.js` (team lead CRUD + approval scoping), `index.html` (sidebar visibility, My Practice scoping, approval access, Team Lead Management UI/modal for SPOCs, export updates), `admin.html` (role dropdowns, cleanup on role change). (feat/roles+db+ui)

- 2026-04-15 (claude) — **AI Prompt Improver (admin-only)** — New "Improve with AI" workflow on the admin Prompt Library page. Admin enters a raw English prompt; OpenAI (`gpt-4o-mini`) qualifies it into a structured prompt (ROLE/TASK/CONTEXT/INPUT/FORMAT/CONSTRAINTS), then Anthropic (`claude-sonnet-4-5`) refines it. Admin reviews side-by-side output and chooses to save to `prompt_library` or discard. Backend: new Supabase Edge Function `prompt-improver` (`supabase/functions/prompt-improver/`) holds API keys as secrets (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`) and re-verifies admin role via the caller's JWT against `public.users.role`. Client uses the user session token; no API keys ever reach the browser. (feat/admin)

- 2026-04-15 (copilot) — **SPOC IDE Usage Stats Page** — New standalone page (`grafana-stats.html`) showing Grafana Copilot IDE usage stats scoped to the SPOC's practice users. SPOCs see only their practice; admins see all with a practice filter. Features: 6 KPI summary cards (total users, avg days active, interactions, generations, acceptances, LOC added), sortable/filterable user table with sparkline bars, data period banner, CSV export. Added `fetchGrafanaStats(practice)` to `db.js`. Navigation link added to sidebar under Management section for admin/spoc roles. (feat/ui)

- 2026-04-15 (copilot) — **SPOC Data Period shows months only** — Changed the Data Period display on the IDE Usage Stats page to show month-only format (e.g. "Feb 2026 to Mar 2026") for SPOC users instead of full dates. Applies to both the banner and CSV export. Admin view retains full date precision. (fix/ui)

- 2026-04-15 (copilot) — **IDE Usage Stats moved into main dashboard** — Moved IDE Usage Stats from standalone `grafana-stats.html` into `index.html` as an inline page section (`page-ide-usage`), eliminating the separate page redirect. Navigation now uses `data-page="ide-usage"` like all other sections. Root `grafana-stats.html` redirects to `index.html#ide-usage`. All features preserved: KPIs, sortable table, practice filter, activity filter, search, CSV export, data period banner with month-only format. (refactor/ui)

- 2026-04-14 (copilot) — **IDE Usage Analytics Page in Admin Panel** — Added new "IDE Usage" page to admin panel showing Grafana Copilot IDE telemetry data. Features: 6 KPI summary cards (active users, code generations, acceptances, LOC added, agent days, chat days), filterable/sortable table with 13 columns, practice filter, search by name/username, CSV export. Data queried live from `copilot_users` IDE columns via Supabase. Accessible at `admin.html#ide-usage`. (feat/ui)

- 2026-04-14 (copilot) — **Fix: Task Count Mismatch — Show Both Total & Approved Counts** — Dashboard KPI "Total Tasks" sourced from `practice_summary` (approved-only) while nav badge and All Tasks tab counted all tasks regardless of approval status. Updated all locations to show both: Dashboard KPI now shows total tasks with "X approved" subtitle, nav badges show `N (M ✓)`, All Tasks pagination shows `(X approved)`, and My Tasks badge includes approved count. Applied same fix to admin.html. (fix/ui)

- 2026-04-15 (copilot) — **Community Prompt Library: Submit + Like/Dislike Voting** — Any authenticated user can submit prompts to the library (immediately visible). Like/dislike voting with toggle support. Prompts with ≥10 likes highlighted as "🔥 Popular". Prompts with ≥10 dislikes hard-deleted. New `prompt_votes` table, 3 SECURITY DEFINER RPCs (`vote_prompt`, `get_prompt_vote_counts`, `add_community_prompt`), updated RLS on `prompt_library`. New modal "Submit a Prompt" with role/category/text fields. Author name displayed on community prompts. (feat/ui+db)

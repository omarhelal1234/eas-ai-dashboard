- 2026-04-27 (claude) — **Fix: prevent duplicate task on rapid double-click of "Save Task".** `saveTask()` in `src/pages/index.html` was an `async` function with multiple `await` points (cap validation, `Phase8.submitWithApproval`) and the modal stayed open during the network insert, so a second click within ~1s entered `saveTask()` again in parallel and both calls reached the insert — producing a duplicate row. Added a module-level `_savingTask` re-entrancy flag (set after sync validation passes, cleared in `finally`) plus button disable + "Saving…" text for visual feedback. Sync validation early-returns leave the flag clear so users can correct errors and retry. `saveAccomplishment()` is structurally similar but closes its modal before the await, so the same race is much less reachable there — left untouched. No cache-buster bump (the fix lives in inline JS in `index.html`; users need a hard reload). (src/pages/index.html)

- 2026-04-14 (copilot) — **Fix: Task/Accomplishment Update Silently Failing (Critical)** — Fixed a column name mismatch that caused ALL non-admin task and accomplishment updates to silently fail. Code referenced `submitted_for_approval` but the actual DB column is `submission_approved`. This also broke approval_id linkage after new inserts — tasks were never linked to their approval records. Additionally fixed check-constraint violation: the code was writing workflow states (`spoc_review`, `admin_review`) into `tasks.approval_status` which only allows `pending`/`approved`/`rejected`. Workflow states now stay in `submission_approvals` only; `approveSubmission()` maps intermediate states to `pending` on the parent table. Fixed in `db.js` (8 occurrences), `ide-task-log/index.ts` (1 occurrence). Added migration `sql/013_fix_column_name_and_linkage.sql`. (fix)

- 2026-04-14 (copilot) — **Fix: Duplicate-on-Edit Bug (Critical)** — Fixed a critical bug where editing any task, accomplishment, copilot user, or issue would create a duplicate instead of updating the existing record. Root cause: `closeModal()` cleared `_editingId`/`_editingType` before the save functions checked them. Fix captures editing state into local variables before closing the modal. Applied to all 5 save functions: `saveTask()`, `saveAccomplishment()`, `saveCopilotUser()`, `saveIssue()`, `saveProject()`. Also fixed orphaned `submission_approvals` — old approval records are now deleted before creating new ones during edits. Added cleanup migration `sql/012_cleanup_duplicates.sql` to remove existing duplicate rows and orphaned approval records. (fix)

- 2026-04-14 (copilot) — **SPOC Mandatory Approval Flow (AI → SPOC → Admin)** — Rewrote the approval routing state machine so that SPOCs are mandated to approve all tasks from their practice members. New flow: (1) AI validates first (`ai_review`), (2) SPOC always reviews next (`spoc_review`), (3) Admin reviews only if saved hours ≥ 15 (`admin_review`). `approveSubmission()` now advances through states instead of jumping directly to `approved`. Updated tier display in task form, toast messages, approval status badges (added `admin_review` / `👑` state), and `handleApprovalAction()` to show contextual messages per stage. (feat)
